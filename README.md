<a name="english"></a>
## English

A full-stack demonstration project that automatically generates functional CRUD (Create, Read, Update, Delete) interfaces by consuming JSON Schemas directly from a .NET 9 backend. This project compares multiple industry-standard libraries (RJSF, Uniforms) across different API protocols, starting with **REST**.

### 🚀 Key Features

- **Multi-Library Comparison**: Compare powerful approaches for schema-to-form generation:
    - **REST + Refine + RJSF**: High-fidelity forms using `react-jsonschema-form` with explicit React Query/Axios usage.
    - **REST + Refine + Uniforms**: Performant and customizable forms using `uniforms` + `ajv`.
- **Dynamic Schema API**: Generic controllers (`RjsfSchemaController`, `UniformsSchemaController`) that generate JSON Schemas for any entity on-the-fly via URL parameters (e.g., `/api/schema/rjsf/Product`).
- **Auto UI Generation**: Utilizes .NET 9 `JsonSchemaExporter` to convert C# entity models into JSON Schema, enabling automatic form generation without manual coding.
- **Modern Data Management**: Integrated with **Refine**, providing a robust data provider layer, notification system, and routing.
- **Dynamic Schema Exploration**: Built-in `SchemaFetcher` tool to inspect raw JSON schemas generated by the backend, categorized by library requirements.
- **Organized Architecture**: Cleanly separated component structure by protocol (REST/GraphQL) and library (RJSF/Uniforms).

### 🛠 Tech Stack

#### Backend
- **Framework**: .NET 9 (ASP.NET Core)
- **ORM**: Entity Framework Core 9.0
- **Database**: PostgreSQL
- **Schema Export**: `System.Text.Json.Schema` (JsonSchemaExporter)

#### Frontend
- **Framework**: [Refine](https://refine.dev/) (Enterprise-grade CRUD framework)
- **UI Kit**: Material UI (MUI) v6
- **Form Libraries**:
    - [react-jsonschema-form (RJSF)](https://rjsf-team.github.io/react-jsonschema-form/)
    - [Uniforms](https://uniforms.tools/) (with AJV validation)
- **Data Fetching**: TanStack React Query v5, Axios

<a name="한국어"></a>
## 한국어

백엔드(.NET 9)에서 제공하는 JSON Schema를 직접 소비하여 CRUD(생성, 조회, 수정, 삭제) 인터페이스를 자동으로 생성하는 풀스택 데모 프로젝트입니다. 이 프로젝트는 **REST**를 시작으로 다양한 API 프로토콜 환경에서 RJSF, Uniforms 등의 산업 표준 라이브러리 통합 방식을 비교합니다.

### 🚀 주요 기능

- **라이브러리 비교 분석**: 스키마 기반의 두 가지 주요 폼 생성 방식을 비교합니다:
    - **REST + Refine + RJSF**: `react-jsonschema-form`을 사용한 고충실도 구현 (React Query/Axios 직접 사용 예제 포함).
    - **REST + Refine + Uniforms**: `uniforms`와 `ajv`를 사용한 고성능 및 커스텀 최적화 구현.
- **동적 스키마 API**: 범용 컨트롤러를 통해 URL 파라미터로 엔티티 이름을 전달받아 즉석에서 JSON 스키마를 생성합니다 (예: `/api/schema/rjsf/Product`). 새로운 테이블 추가 시 컨트롤러 수정이 필요 없습니다.
- **자동 UI 생성**: .NET 9의 `JsonSchemaExporter`를 사용하여 C# 엔티티 모델을 JSON Schema로 변환하고, 이를 기반으로 코딩 없이 UI를 자동 생성합니다.
- **현대적인 데이터 관리**: **Refine** 프레임워크를 기반으로 강력한 데이터 프로바이더, 알림 시스템, 라우팅 기능을 제공합니다.
- **동적 스키마 탐색**: 백엔드에서 생성된 원본 JSON 스키마를 라이브러리별 요구사항에 맞춰 실시간으로 확인할 수 있습니다.
- **체계적인 아키텍처**: 프로토콜(REST/GraphQL) 및 라이브러리(RJSF/Uniforms)별로 컴포넌트를 분리하여 확장성을 확보했습니다.

### ⚙️ Getting Started / 실행 방법

#### 1. Database Setup
Run PostgreSQL using the provided `docker-compose.yml` in the root directory.
```bash
docker-compose up -d
```

#### 2. Running the Backend
- **Visual Studio**: `ReactFormAutoGenerate.Server`를 시작 프로젝트로 설정 후 **F5**.
- **CLI**: 
  ```bash
  cd ReactFormAutoGenerate.Server
  dotnet run
  ```

#### 3. Running the Frontend
```bash
cd ReactFormAutoGenerate.Client
npm install
npm run dev
```
